Archlinux Post-Install !

14-09-2020 — Ecrit par AlrikRr — 7 Minutes

Post installation Archlinux

Je vous mets à disposition mes fichiers de configuration à la fin de ce tuto.

Bonne configuration :)

Clavier

Votre clavier n’est pas en français ?

$ keymaps fr
OU
$ setxkbmap fr

Pour changer la langue de manière permanante :

$ sudo vim /etc/locale.gen

et dé-commentez la ligne :

#LANG=fr_FR.UTF-8

Voir : Clavier Archlinux Wiki

Networking

Impossible de ping ?

Jettez un oeil à votre /etc/resolv.conf et ajoutez la ligne suivante

nameserver 1.1.1.1

Ethernet

Si vous n’arrivez toujours pas à accéder à internet :

systemctl start systemd-networkd

Récupéré le nom de votre interface ethernet ip a et créez le fichier suivant :

vim /etc/systemd/network/<interface>.network

Ajoutez :

[Match]
Name=<interface>
[Network]
DHCP=ipv4

Wi-Fi

Pour trouver le nom de la carte wifi :

$ lscpi

De mon côté, j’ai une carte Qualcomm Atheros QCA9377, je vais avoir besoin des drivers ath10k_pci

sudo pacman -S NetworkManager
sudo systemctl enable NetworkManager
sudo systemctl start NetworkManager
sudo pacman -S linux-firmware
sudo modprobe -r ath10k_pci && sudo modprobe ath10k_pci

si la carte wifi n’est pas UP :

sudo ip link set wlan0 up

Certaines cartes peuvent être soft-bloquées. Pour les débloquer, utiliser l’outil rfkill :

sudo pacman -S rfkill

Pour afficher la liste des device bloqués :

rfkill

Ensuite pour les débloquer :

sudo rfkill unblock all

Power Management

Pour la gestion de l’alimentation, il nous faut installer un outils pour gérer les différents profils d’alimentation.

Un problème assez embêtant sur la configuration actuelle c’est la carte wifi qui se désactive quand le laptop est fermé.

Pour résoudre le problème il nous faut installer et configuer tlp :

sudo pacman -S tlp tlp-rdw

Ensuite on initialise nos services :

systemctl enable tlp.service
systemctl enable NetworkManager-dispatcher.service
systemctl mask systemd-rfkill.service
systemctl mask systemd-rfkill.socket

Ensuite pour la configuration, le fichier se trouve dans /etc/tlp.conf. Il vous faut modifier les lignes suivantes :

WIFI_PWR_ON_AC = on
WIFI_PWR_ON_BAT = on

Update

Pour voir si tout est ok, on va lancer une update :

pacman -Syu

Si tout est ok, on peut continuer, sinon je vous invite à rechercher vos messages d’erreurs ou vos equipement réseau sur internet et de fouiller sur les forums Archlinux :)

AUR yay

Pour téléchargez les paquets AUR de archlinux, nous allons utiliser [YAY](# AUR yay)

$ git clone https://aur.archlinux.org/yay.git
$ cd yay
$ makepkg -si

Pour installer un paquet, c’est comme pacman :

$ yay -S <aur>

Interface graphique

fonts

Les fonts sont nécessaire pour certaines applications afin d’afficher des emojis et autres langues différentes :

Les fonts que j’utilise pour ma polybar

$ sudo pacman -S noto-fonts-cjk noto-fonts-emoji noto-fonts
$ yay -S nerd-fonts-complete
$ yay -S ttf-freefont ttf-ms-fonts ttf-linux-libertine ttf-dejavu ttf-inconsolata ttf-ubuntu-font-family

Pour recharger la configuration des fonts :

$ fc-cache -vf

I3-gaps

Pour l’interface graphique, je vais installer i3-gaps et Xorg-server.

$ sudo pacman -S i3-gaps ttf-dejavu i3status i3blocks i3lock
$ sudo pacman -S xorg xorg-init terminator
$ sudo pacman -S xf86-video-intel mesa 
$ pacman -S alsa-utils

Modifiez le fichier suivant :

$ vim /etc/X11/xinit/xinitrc

Commentez :

# twm &
# exec xterm

Ajoutez :

exec i3

Créer un dossier i3 das .config à la racine de votre home :

$ mkdir ~/.config/i3

Copier la configuration par défaut de i3 :

cp /etc/i3/config ~/.config/i3/config

Maintenant, on va démarrer notre interface graphique :

startx

Chemin du fichier : ~/.config/i3/config

Mon fichier de conf : config

lightdm

Il nous faut installer lightdm pour choisir notre desktop et avoir une interface de connexion au démarrage.

$ sudo pacman -S lightdm lightdm-gtk-greeter lightdm-gtk-greeter-settings

$ systemctl enable lightdm

$ reboot

Pour éditer le theme lightdm-greeter, lancer l’application en root :

$ sudo lightdm-gtk-greeter-settings

polybar

Pour avoir la documentation complète, rendez-vous ICI.

Une liste de themes ICI.

yay polybar

Pensez à copier le fichier de configuration par défaut :

$ cp /usr/share/doc/polybar/config /home/eidorian/.config/polybar/config

Créez le script à l’emplacement suivant ~/.config/polybar/launch.sh

#!/bin/bash

# Terminate already running bar instances
killall -q polybar

# Wait until the processes have been shut down
while pgrep -u $UID -x polybar >/dev/null; do sleep 1; done

# Launch Polybar, using default config location ~/.config/polybar/config
polybar mybar &

echo "Polybar launched..."

Et ajoutez la ligne suivante dans la configuration i3 :

exec_always --no-startup-id $HOME/.config/polybar/launch.sh

Ne pas oublier de changer [bar/exemple] en [bar/mybar] dans le fichier de configruation de polybar.

Chemin du fichier : ~/.config/polybar/config

Mon fichier de conf : config

Pour plus d’information sur polybar : rendez-vous ICI

rofi

rofi va nous service de lanceur d’application.

Installation :

$ sudo pacman -S rofi

Modifier le fichier de configuration i3

vim ~/.config/i3/config

Modifier le bindsym associé par celui-là :

bindsym Mod4+d exec rofi -modi drun -show drun -font "DejaVu Sans 15" -show-icons -theme slate

Chemin du fichier :~/.config/rofi/slate.rsi

Mon fichier de conf :config

feh

feh va nous servir afin d’afficher un fond d'écran avec i3.

Pour l’installer :

$ sudo pacman -S feh

On ajoute ensuite cette ligne dans le fichier de configuration i3 :

exec_always --no-startup-id feh --bg-fill --randomize ~/Images/wallpaper/*

i3-gaps

Pour l’utilisation de i3-gaps, on va ajouter les lignes suivantes dans le fichier de configuration i3 :

gaps inner 10
gaps outer 20
smart_gaps on
for_window [class="^.*"] border pixel 0

compton et picom

Pour la transparence du terminal j’utilise picom, anciennement compton. Mon terminal est Terminator

installation :

sudo pacman -S compton

Ancienne version (ne fonctionne plus):

Pour la configuration, on ajoute cette ligne dans le fichier de configuration i3:

exec_always --no-startup-id compton --config /

On va ensuite modifier le fichier de configuration compton, s’il n’existe pas, on va le créer:

vim ~/.config/compton.config

On ajoute :

opacity-rule = ["90:class_g = 'Terminator'"]

Nouvelle version :

Dans la config i3 :

exec_always --no-startup-id ~/.config/polybar/scripts/picom.sh

Le script picom.sh :

vim .config/polybar/scripts/picom.sh

#!/bin/bash
killall picom
picom -b

Il ne nous reste plus qu'à modifier la transparence directement avec les options du terminal.

neofetch

Afficher les information du système dans le terminal :

$ sudo pacman -S neofetch

Chemin du fichier :~/.config/neofetch/config.conf

Mon fichier de conf :config

flameshot

Un outil très puissant pour les capture d'écran, on va le blind avec un raccourci i3, la touche impr imprime-écran.

$ sudo pacman -S flameshot

On ajoute ensuite cette ligne dans la config i3 :

bindsym --release Print exec flameshot gui

Outils Binaire

Pour l’exploitation de binaire et le reverse, je vais devoir installer des librairies de la libc ainsi que d’autres outils :

sudo pacman -S gdb lib32-glibc  gcc-multilib lib32-gcc-libs

Pour l’installation de gef , le plugin GDB :

$ wget -O ~/.gdbinit-gef.py -q http://gef.blah.cat/py
$ echo source ~/.gdbinit-gef.py >> ~/.gdbinit

Dispo sur le Github ICI

Serveur Samba

Pour monter un serveur samba qui se trouve sur votre réseau, il vous faut les outils cifs :

sudo pacman -S cifs-utils

Une fois les outils installé, il vous faut redémarrer votre ordinateur et vérifier que le module kernel est bien chargé :

sudo modprobe cifs

Bien, maintenant, il vous reste plus qu'à monter votre partage :

sudo mount -v -t cifs //192.168.1.100/public /home/eidorian/Public -o user=guest

sudo  modprobe vboxdrv

Arduino

sudo usermod -G lock eidorian

sudo modprobe cdc_acm
