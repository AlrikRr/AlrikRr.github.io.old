K-Nearest Neighbors

2020-06-18 â€” Written by Eidorian

If you want a better tutorial, I recommend you to follow TechWithTim I’ll try to explain the theory in a simple way, i’m not an expert, this tutorial is also benefits for me to really understand the logic.

We assume we have three classes, RED, BLUE and GREEN.
But we don’t know the class of the BLACK one.

Human Style :

If you are a human, well at least I hope so, you can definitely put the BLACK one with the RED class because it is the closest than the other classes.

Robot Style :

Now if you are less human, you will require the K parameter, it represents the number of neighbors that we are looking for to class the BLACK point.

As we can see, the 3 nearest point are going to vote, two RED and one GREEN so the BLACK point will be RED. Simple.
BUT now imagine if the K parameter is 4 and then you have 2 RED and 2 GREEN. What do we conclude?
Well, you just definitely don’t set even number because we need a winner!

Let’s explain the details of how the computer calculates the distance between the points and found the nearest neighbors.
As you can see, this is an Euclidian Distance, we need to determine the best magnitude, also the distance from each point of the BLACK point.

KNN works with multiple Plan, here 3D plan example.
Now let’s see an error of the utilisation of KNN. In this example, the K number is excessively high. When we look at this 3D plan, the BLACK point is more likely to belong to the RED points but the K parameter is nine, there is only 4 RED points and 5 BLUE points.
At the end, the BLUE win because they are more numerous than the RED.
Therefore if you want to select a K parameter, be careful and don’t use unnecessarily high value or the result will be compromise.

KNN limitations must be taken into consideration. The computer will calculate the distance for ALL the data point on the plan to find the nearest. Imagine if there are 100,000,000 data points!
This is why it is useless to train this algorithm, because it needs calculating the distance each time we run it.
