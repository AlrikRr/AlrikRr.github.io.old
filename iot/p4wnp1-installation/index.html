#1 P4wnP1: ALOA Installation

11-11-2020 — Ecrit par AlrikRr — 3 Minutes

Hello ! Aujourd’hui nous allons mettre en place le couteau suisse du pentester.

Je parle du projet P4wnP1 ALOA de RoganDawes!

Initialiement, le projet P4wnP1 vient de MaMe82.Je vais ici vous montrer comment installer la version A.L.O.A (A Little Offensive Appliance) sur votre RaspberryPi Zero !

La version ALOA est un peu plus simple à configurer et possède une interface graphique via un serveur web pour la configuration des payloads.

Qu’est ce que c’est ??

P4wnP1 fonctionne comme une backdoor accessible à distance via un poitn d’accès WI-Fi.
Il vous suffit la brancher sur l’ordinateur de votre victime et le tour est joué.
Comme une RubberDucky ou une BashBunny, P4wnP1 est reconnu comme un clavier par l’ordinateur ce qui lui laisse la possibilité d’executer à peut prêt tout ce que vous pouvez :)

  • Execution de commande comme un clavier
  • Execution de mouvement comme une souris
  • Se faire passer pour une clé USB pour exfilter des données
  • etc …

La liste est longue, vous retrouverez beaucoup plus d’exemple d’attaques sur le github !

Attention : Il faut savoir que ce n’est pas une RubberDucky, la P4wnP1 est plus lente, si vous la branchez sur un ordniateur avec un script d’execution de code comme un clavier, il faut attendre de l’OS, qui est kali linux, finisse de charger avant de pouvoir executer ses commandes. Ou alors vous avez prévu le coup et branché votre P4wnP1 sur une batterie externe avec le port d’alimentation ;)

Passons maintenant à l’installation !

Prérequis:

  • Une carte SD (16GB)
  • Une carte Raspberry Pi Zero w
  • Un lecteur de carte SD USB
  • Client SSH (Terminal linux ou Putty pour windows)
  • BalenaEtcher pour flasher la carte SD

Flash de l’ISO

Vous trouverez le fichier Image dans la section releases sur le github.

Pour cette installation, j’utilise la version v.0.1.1-beta.

Installez BalenaEtcher, l’avantage c’est que l’outil est multi plateformer (Windows, Linux et MaxOS).
Sinon, libre à vous d’utiliser l’outil que vous voulez.

Une fois le fichier img copié sur la carte SD, on peut passer à l'étape suivante !

Branchement de la raspberry

Pour le branchement, faites attention, il faut utiliser le port USB de la carte RaspberryPi Zero afin de pouvoir transferer des données à la machine qui est connectée.

Faites attention au câble que vous utilisez, certains ne permettent pas le transfert de données. Malheureusement il n’y a aucun moyens de les différencier.

# Connexion au Wi-Fi

Une fois votre raspberry branché, vous dervriez voir un point d’accès wifi assez explicit.

Le mot de passe ce trouve sur le github : MaMe82-P4wnP1

Interface Web

Une fois connecté , attention vous n’aurez plus accès à internet si vous utilisé la même carte Wi-Fi que celle que vous utilisez.

Rendez-vous sur http://172.24.0.1:8000

Vous avez accès à pas mal de menu, ici celui qui va nous intéressé c’est le menu Wi-Fi Settings.
Je vais vous montrer comment fonctionne la configuration sur P4wnP1 en prenant l’exemple du Wi-Fi.

Ici vous pouvez paramétrer quelques options comme :

  • Le channel de diffusion de notre point d’accès Wi-Fi
  • Le mode d’authentification
  • Le SSID affiché
  • SI le SSID doit être caché
  • Pour finir, le mot de passe du point d’accès.

Si vous changez quelques paramètres, il vous faut ensuite déployer vos modification avec le bouton DEPLOY

  • DEPLOY : Permet de déployer les modification sur votre P4wnP1
  • DEPLOY STORED : Permet de déployer une configuration déjà enregistrée
  • RESET : Applique la configuration par défaut
  • STORE : Permet de stocké une configuration pour l’utiliser plus tard
  • LOAD STORED : Permet d’appliquer une configuration enregistrée dans les champs.

Connexion SSH

Vous avez la possiblité de vous connecter à votre Raspberry via SSH en vous connectant avec le compte :

  • root
  • toor

Hack !

Restez connecté ;)
