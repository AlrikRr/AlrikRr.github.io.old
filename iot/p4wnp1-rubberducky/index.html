#2 P4wnP1: RubberDucky

18-11-2020 — Ecrit par AlrikRr — 3 Minutes

Hello !

Aujourd’hui nous allons voir la configuration de notre P4wnP1 pour en faire une RubberDucky !

Je vous invite à regarder l’installation de P4wnP1 si cela n’est pas encore fait !

Comme je l’ai dit dans le tuto #1, P4wnP1 est beaucoup plus lent qu’un RubberDUcky classic. Si vous ne le branchez pas sur une batterie externe, votre script ne s’executera qu’après le démarrage de l’OS sur votre P4wnP1.

Configuration P4wnP1

Par défaut notre P4wnP1 est déjà configuré pour executer des commandes RubberDucky.

Comme vous pouvez le voir, vous avez aussi la possibilité d'écrire des scripts permettant de simuler une souris.

Vous pouvez également personnalisé totalement votre P4wnP1 pour que celle-ci soit reconnu comme le périphérique de votre choix par l’ordinateur de votre victime.

Pour l'écriture de nos scripts, c’est par ici !

Scripting

Le script par défaut devrait fonctionner sous windows sans aucun problème.
Mais il y a quelques modifications à apporter.

Le language utilisé ici est du JavaScript ce qui laisse beaucoup plus de possibilité qu’un simple language DuckyScript !

Langue du clavier

layout('us');

Par défaut, le script utilise un clavier US, si vous utilisez un clavier FR en AZERTY, il vous faut changer la langue du clavier.

layout('fr');

Delay

delay(1000);

Voici un delay d’une seconde.

Les delay sont très important dans un script RubberDucky.
Si vous mettez des delay trop rapide, votre script risque de louper des commande et tout faire foirer.
Suivant la puissance de l’ordniateur, il vous faudra adapter les delay. Le mieux est de mettre des delay trop lent que trop court.

Quelques commandes JavaScript

Voici quelques commandes qui peuvent vous être utile pour vos scripts :

  • type("Hello, Friend."); Permet de saisir du texte.
    • TIPS : Ajoutez un \n à la fin pour simuler la touche ENTER.
  • press("ENTER"); Permet d’appuyer sur une touche, ici la touche ENTER (Entrée).

Quelques commandes DuckyScript

Vous l’avez compris, les commandes DuckyScript seront utilisé dans les fonctions press().
En voici quelques exemples :

  • GUI ou WINDOWS Simule la touche Windows
  • SHIFT Simule la touche SHIFT
  • ALT Simle la touche ALT
  • CONTROL ou CTRL Simule la touche Ctrl
  • REPEAT n Répète la dernière commande n fois
  • CAPSLOCK Simule la touche Majuscule
  • ESC ou ESCAPE Simule la touche Echape
  • SPACE Simule la touche Espace
  • TAB SImule la touche Tabulation
  • Pour les autres, voici une petite Cheatsheet

Execution de notre script

ATTENTION aux commandes que vous allez écrire, je ne serais tenu responsable des dégats causé sur votre ordinateur !

Pour executer votre script RubberDucky, il vous suffit de cliquer sur le bouton RUN

Voici mon script compatible pour mon Linux.
J’ai du mettre des delay un peu partout car j’avais quelques soucis de latence.

layout('fr');
delay(1000);
press("CONTROL ALT t");
delay(2000);
type("htop");
delay(2000);
press("ENTER");
delay(2000);
press("GUI");
delay(2000);
type("editeur");
delay(2000);
press("ENTER");
delay(2000);
type("Hello, friend.");
